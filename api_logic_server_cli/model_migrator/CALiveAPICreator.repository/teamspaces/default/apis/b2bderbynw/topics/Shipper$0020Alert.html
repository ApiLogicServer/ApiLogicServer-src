Business Background

 

Story: Place Order

As a Business Owner, 

I want to enable Partners to Place Orders via RESTful APIs and Web/Mobile Appis,

So that they can self-serve to reduce their / our Adminstration Costs, and reduce delays.

 

Acceptance Criteria: Shipper Alert

Given a Shipper that has reqistered a Webhook (in shipper.webHookURL),

When an order is placed for a registered Shipper (identified by the Orders ShipTo),

Then ensure that the system Posts a message to the Shipper in an agreed-upon API format (per ShipperAPIDef).

 

Technical Approach:

Some orders have a ShipTo, denoting who ships the order.  Shippers can register for WebHooks, by supplying a URL stored in Shipper.OrderChangeWebHook.

Tech approach: Commit Event, invoking B2B.transformToWebHook
