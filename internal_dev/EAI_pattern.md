## Pattern: Two-Message Kafka Design

**Flow:**
1. `isdc` topic → insert `ShipmentXml` row → `row_event` fires → publish to `shipment_processed`
2. `shipment_processed` topic → parse XML → persist `Shipment` + related rows

**Why 2 messages (not 1):**
SQLAlchemy: a failed flush poisons the session — you cannot commit the blob in the same session
after a parse exception. Fixing it requires 3 explicit back-to-back transactions + session gymnastics.
Kafka acts as the durable commit boundary between ingestion and processing:
- Tx1: always saves XML blob (guaranteed)
- Tx2: parse + persist (failure isolated, no session contamination)
- Back-pressure decoupling is a free bonus

**Key files:**
- `integration/kafka/kafka_consumer.py` — `isdc` + `shipment_processed` handlers
- `logic/logic_discovery/shipment_xml_ingest.py` — `row_event` bridge (insert ShipmentXml → publish)
- `integration/CIMCorpShipmentMapper.py` — XML → ORM mapper

## Pattern: Generic XML/DB Mapper

**Current location:** `integration/CIMCorpShipmentMapper.py`

**Algorithm (`populate_row`):**
1. For each child XML element: strip namespace → local name
2. Check `exceptions` dict: `None` = skip, `str` = remap to that column name
3. Fallback: `lowercase(xml_name)` → column name
4. If column exists on model (via `sa_inspect`): `setattr(row, col, coerced_value)`
5. Apply `overrides` dict last (forced values regardless of XML)

**Defensive:** only sets columns that exist on the model — survives schema drift.

**Proposed split for prototype base + CE:**

### `integration/system/xml_mapper.py` (goes in `prototypes/base`)
Generic, no schema knowledge:
```python
def populate_row(row, element, exceptions: dict = None, overrides: dict = None):
    """
    Map XML element children → ORM row columns by name (lowercase match).
    exceptions: {xml_name: col_name | None}  — None = skip, str = remap
    overrides:  {col_name: value}            — forced values applied last
    """
```
Plus private helpers: `_coerce()`, `_local()`, `_column_names()`.

## Performance Notes (relevant for FedEx load testing)

- `log()` / `log_engine()` in LogicBank call `str(self)` (full row serialization) unconditionally
- At batch scale (10k messages × 50 log calls × 10μs each = ~5s) even with logging OFF
- Fix: add `isEnabledFor(logging.INFO/DEBUG)` guard before string work in both methods
- Python 3.13 zero-cost exceptions: `try/except` with no raise is essentially free — no overhead concern for the multi-constraint `_constraints()` change
